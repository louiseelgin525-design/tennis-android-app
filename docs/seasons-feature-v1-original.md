# Сезоны клуба — техническое задание для Antigravity

## 1. Цель

Добавить в существующее приложение клубные сезоны, не ломая уже работающий функционал.

Сейчас в приложении уже есть:

- база игроков;
- общий рейтинг клуба;
- создание и проведение турниров;
- перерасчёт рейтинга после турнира;
- история турниров;
- профили игроков;
- награды;
- рекорды клуба.

Нужно добавить **сезонный рейтинг** как отдельный слой поверх существующей системы.

Главный принцип:

> Общий рейтинг продолжает работать как раньше и никогда не обнуляется.  
> Сезонный рейтинг существует отдельно и в начале каждого нового сезона стартует с одинакового значения для всех игроков.

---

## 2. Что нельзя ломать

Необходимо сохранить без изменения поведения:

- экран «База игроков»;
- поиск игроков;
- сортировку по рейтингу и алфавиту;
- карточки игроков;
- общий рейтинг;
- существующий алгоритм общего рейтинга;
- создание турнира;
- выбор участников;
- проведение турнира;
- завершение турнира;
- сохранение результатов;
- историю турниров;
- существующие награды;
- существующие рекорды;
- текущую структуру профиля игрока;
- существующие маршруты, если их можно сохранить;
- работу приложения без активного сезона.

Экран «База игроков» должен остаться базой игроков.  
Сезон нельзя помещать внутрь списка игроков или делать переключателем «Общий / Сезон».

Желательно сохранить существующий технический маршрут раздела «Рекорды», чтобы не ломать навигацию и ссылки. Меняется прежде всего название и содержимое экрана.

### Назначение разделов

#### Рейтинг

Оставить без изменений:

- база игроков;
- общий рейтинг;
- поиск;
- сортировка;
- постоянные награды игроков.

#### Турнир

Оставить без изменений.

#### История

Оставить без изменений.

#### Клуб

Здесь находятся:

- текущий сезон;
- лидер сезона;

## 4. Логика рейтингов

У игрока одновременно существуют два независимых рейтинга.

### 4.1 Общий рейтинг

Это текущий рейтинг клуба за всё время.

Требования:

- не обнуляется;
- не пересчитывается заново при старте сезона;
- продолжает работать по существующему алгоритму;
- отображается в базе игроков и профиле игрока;
- участвует во всей старой логике приложения.

### 4.2 Сезонный рейтинг

Это отдельный рейтинг только для активного сезона.

Пример сезона:

- название: `Осень 2026`;
- дата начала: `01.09.2026`;
- дата окончания: `01.01.2027`;
- стартовый рейтинг: `100.0`.

На старте сезона все игроки получают одинаковый сезонный рейтинг:

```text
100.0
```

После каждого завершённого турнира:

1. Общий рейтинг рассчитывается как раньше.
2. Отдельно рассчитывается сезонный рейтинг.
3. Для сезонного расчёта используются текущие сезонные рейтинги игроков.
4. Общий рейтинг не используется как стартовое значение сезонного расчёта.
5. История сезонных изменений сохраняется отдельно.

Важно:

> Нельзя просто копировать изменение общего рейтинга в сезонный рейтинг.

Следует использовать тот же принцип расчёта рейтинга, но передавать в него другой набор входных данных.

Условно:

```text
calculateGeneralRating(results, generalRatings)
calculateSeasonRating(results, seasonRatings)
```

## 5. Жизненный цикл сезона

### 5.1 Статусы

У сезона должны быть состояния:

```text
draft
active
completed
```

Одновременно у клуба может быть только один активный сезон.

### 5.2 Создание сезона

Создавать сезон может только администратор клуба.

Поля:

```text
Название
Дата начала
Дата окончания
Стартовый рейтинг
```

Значение стартового рейтинга по умолчанию:

```text
100.0
```

### 5.3 Старт сезона

При запуске сезона:

- создать сезонную запись для каждого существующего игрока клуба;
- присвоить каждому сезонный рейтинг `100.0`;
- обнулить только сезонную статистику;
- не менять общий рейтинг;
- не менять общую историю;
- не удалять старые награды;
- не менять старые турниры.

### 5.4 Новый игрок во время сезона

Если игрок добавлен после начала сезона:

- создать ему сезонную запись;
- присвоить сезонный рейтинг `100.0`;
- включить его в сезон после первого участия в сезонном турнире;
- общий рейтинг оставить независимым.

### 5.5 Завершение сезона

После завершения:

- сезонный рейтинг перестаёт изменяться;
- итоговая таблица фиксируется;
- игрок с максимальным сезонным рейтингом становится чемпионом сезона;
- топ-3 сохраняется;
- сезонные награды сохраняются;
- сезон становится доступен в истории;
- общий рейтинг остаётся без изменений.

При старте нового сезона все снова начинают сезонный рейтинг со значения `100.0`.

Нужно взять текущий экран «Рекорды клуба», переименовать его в **«Клуб»** и мягко добавить сверху блок текущего сезона.

Все существующие блоки необходимо сохранить.

Кнопку «Все награды» сохранить.

## 7. Карточка текущего сезона

Карточка размещается первым блоком экрана «Клуб».

### Содержимое

```text
Сезон Осень 2026

1 сентября — 1 января
До конца: 47 дней

Лидер сезона

[Аватар] Васильев Николай
         126.7
         1 место в сезоне

🥇 Васильев 126.7
🥈 Иванов 118.4
🥉 Петров 111.2

Открыть сезон →
```

### Поведение

- вся карточка кликабельна;
- по нажатию открывается отдельный экран сезона;
- если активного сезона нет, показывается нейтральная заглушка;
- администратор видит кнопку «Создать сезон»;
- обычный игрок видит текст «Новый сезон скоро начнётся».

### UI

Использовать текущую светлую стилистику приложения:

- фон приложения: существующий светло-серый;
- карточка: белая или светло-голубая;
- рамка: тонкая голубая;
- скругления: как у текущих карточек;
- основной акцент: текущий синий;
- золото: только для кубка и первого места;
- не вводить тёмную или фиолетовую тему.

Рекомендуемые размеры:

- боковые отступы: `16 px`;
- внутренний padding: `20 px`;
- радиус: `22–24 px`;
- высота карточки: примерно `220–250 px`;
- минимальная высота кликабельной зоны: `48 px`.

## 8. Новый экран сезона

Рекомендуемый маршрут:

```text
/club/:clubId/season/:seasonId
```

Допустим и более простой вариант:

```text
/season/:seasonId
```

### 8.1 Шапка

```text
← Сезон Осень 2026

1 сентября — 1 января
До конца: 47 дней
```

Кнопка назад возвращает пользователя в раздел «Клуб».

### 8.2 Карточка лидера

```text
ЛИДЕР СЕЗОНА

[Аватар] Васильев Николай

126.7
1 место в сезоне

+4.2 за неделю
```

Ниже четыре показателя:

```text
28 матчей
15 побед
3 турнира
7 побед подряд
```

Все показатели считаются только внутри текущего сезона.

### 8.3 Топ сезона

Строка участника:

```text
[место] [аватар] Имя игрока    126.7   +4.2   >
```

Показывать:

- позицию;
- аватар;
- имя;
- сезонный рейтинг;
- изменение сезонного рейтинга за последние 7 дней;
- переход в профиль игрока.

Оформление мест:

- 1 место — золотой маркер;
- 2 место — серебряный;
- 3 место — бронзовый;
- остальные — обычные серые цифры.

Общий рейтинг здесь не показывать.

### 8.4 Продвижение за неделю

```text
Продвижение за неделю

Васильев Николай    +4.2
Петров Петр         +3.1
Иванов Иван         -1.4
```

Цвета:

- рост — зелёный;
- снижение — красный;
- без изменения — серый.

### 8.5 Награды сезона

Не создавать новую сложную систему наград.

Использовать существующие достижения, но считать их только по данным текущего сезона:

- лучшая серия побед;
- самый активный;
- самый быстрый рост;
- победитель последнего турнира;
- игрок недели.

Кнопка:

```text
Все награды
```

может открывать существующий экран наград с фильтром по текущему сезону.

### 8.6 Поделиться сезоном

Внизу экрана:

```text
Поделиться сезоном
```

На первом этапе достаточно формировать текст:

```text
🏆 Сезон Осень 2026

1 место — Васильев Николай, 126.7
2 место — Иванов Иван, 118.4
3 место — Петров Петр, 111.2
```

Генерацию красивого изображения можно добавить позже.

## 12. Интеграция с завершением турнира

Существующий сценарий завершения турнира должен остаться главным.

Рекомендуемый порядок:

```text
saveTournament()
updateGeneralRating()
updateGeneralAwards()
```

После этого, только если есть активный сезон и турнир входит в его даты:

```text
updateSeasonRating()
updateSeasonStats()
updateSeasonAwards()
```

Важно:

- сезонная ошибка не должна отменять сохранение турнира;
- сезонная ошибка не должна отменять обновление общего рейтинга;
- ошибки сезонной логики нужно логировать отдельно.

Условно:

```text
saveTournament()
updateGeneralRating()

try:
    updateSeasonRating()
except:
    logSeasonError()
```

## 13. Работа без активного сезона

Сезоны должны быть необязательными.

Если активного сезона нет:

- турниры работают как раньше;
- общий рейтинг работает как раньше;
- история работает как раньше;
- профиль игрока работает как раньше;
- сезонный блок в профиле не показывается;
- раздел «Клуб» показывает существующие рекорды;
- вместо карточки сезона показывается заглушка.

Для администратора:

```text
Сейчас активного сезона нет
Создать сезон
```

Для обычного игрока:

```text
Новый сезон скоро начнётся
```

## 14. Поэтапная реализация

### Этап 1. Анализ проекта

Перед изменениями:

- изучить текущую архитектуру;
- найти расчёт общего рейтинга;
- найти обработчик завершения турнира;
- найти хранилище игроков;
- найти экран рекордов;
- найти профиль игрока;
- найти маршрутизацию;
- перечислить файлы, которые будут изменены.

### Этап 2. Данные

- добавить сущность сезона;
- добавить сезонную статистику;
- добавить историю сезонного рейтинга;
- не менять UI;
- не менять общий рейтинг.

### Этап 3. Расчёт

- добавить отдельный расчёт сезонного рейтинга;
- использовать сезонные значения как входные данные;
- добавить тесты;
- убедиться, что общий рейтинг работает как раньше.

### Этап 4. Интеграция с турниром

- запускать сезонный расчёт только при активном сезоне;
- не ломать завершение турнира;
- добавить обработку ошибок.

### Этап 5. Экран «Клуб»

- визуально переименовать «Рекорды» в «Клуб»;
- сохранить текущие рекорды;
- добавить карточку текущего сезона сверху.

### Этап 6. Экран сезона

- продвижение за неделю;

### Этап 7. Профиль игрока

- добавить компактный блок сезона;
- остальной профиль оставить без изменений.

### Этап 8. Управление сезоном

- создание;
- запуск;
- завершение;
- история сезонов.

## 15. Критерии готовности

Функция считается готовой, если:

1. Старые турниры открываются как раньше.
2. Новый турнир можно провести без активного сезона.
3. Общий рейтинг рассчитывается так же, как до изменений.
4. При активном сезоне обновляются оба рейтинга.
5. Все участники нового сезона начинают с `100.0`.
6. Общий рейтинг никогда не сбрасывается.
7. Экран «База игроков» не содержит сезонных карточек.
8. Сезон расположен в разделе «Клуб».
9. Старые рекорды и награды сохранены.
10. Профиль игрока содержит отдельный блок сезона.
11. Завершённый сезон больше не меняется.
12. Новый сезон снова начинается со `100.0`.
13. Ошибка сезонного расчёта не ломает сохранение турнира.
14. Интерфейс корректно работает на мобильных экранах шириной от `360 px`.
15. Приложение без активного сезона работает полностью как раньше.

## 17. Главный принцип

> Не переписывать приложение под сезоны.  
> Добавить сезоны как независимый и обратно совместимый слой поверх работающих турниров, общего рейтинга, профилей, наград и рекордов.

Общий рейтинг живёт в разделе **«Рейтинг»**.

Сезон живёт в разделе **«Клуб»**.

В профиле игрока они отображаются отдельными блоками.
