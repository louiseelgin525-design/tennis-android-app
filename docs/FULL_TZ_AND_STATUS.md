# Полное ТЗ и статус TennisTourney

## Цель

Android-приложение для проведения турниров по настольному теннису.

## Стиль

Apple/iOS-like:
- светлый фон `#F2F4F7`;
- белые карточки;
- Apple Blue `#007AFF`;
- крупные читаемые элементы;
- минимум визуального шума;
- всё удобно на реальном Android-телефоне.

## База игроков

Игрок должен быть зарегистрирован до участия в турнире.

Карточка игрока хранит:
- ФИО;
- текущий рейтинг;
- сколько турниров уже сыграл;
- победы;
- поражения;
- золотые/серебряные/бронзовые медали;
- аватарку;
- статистику с соперниками.

Если `tournamentsPlayed = 6`, правило новичка уже не работает.

## Участие в турнире

Игрок не может участвовать, если его нет в базе.

В создании турнира:
- вводим 2+ буквы;
- выбираем игрока из базы;
- ручной ввод незарегистрированного игрока не должен проходить;
- однофамильцы должны различаться.

## Круговой турнир до 10 игроков

- каждый играет с каждым;
- шахматка;
- active tables;
- ввод результата через ячейку или карточку стола;
- количество столов 1/2/3+;
- W/L для снявшихся игроков;
- W/L не меняет рейтинг.

## Подсчёт мест

- победа = 1 очко;
- поражение = 0;
- при равенстве очков — личные встречи;
- затем отношение партий в личных встречах.

## RTTF-подобный рейтинг

Формула:

```text
Δ = [(100 - (PВ - PП)) / 10] × k × D
```

`k` по среднему рейтингу турнира:

```text
Rср < 250      k = 0.20
250–349        k = 0.25
350–449        k = 0.30
450–549        k = 0.35
550+           k = 0.40
```

`D`:

```text
разница 1 партия  → 0.8
разница 2 партии  → 1.0
разница 3+ партии → 1.2
```

Если победитель выше проигравшего на 100+ рейтинга, изменение = 0.

Рейтинг хранится с десятыми. Рейтинг не ниже 1.

## Новички

Если игрок сыграл не более 5 турниров:
- победа: `k = 1`, `D = 1`;
- поражение: `k = 0.5`, `D = 1`.

Если в базе у игрока `tournamentsPlayed = 6`, новичком он уже не считается.

## Бонусы за медали

После завершения турнира судья выбирает:

```text
Начислить бонусы / Пропустить
```

Мягкие бонусы:
- 1 место: +3;
- 2 место: +2;
- 3 место: +1.

Медали сохраняются в базе в любом случае.

## Экран итогов

- маленькие плашки 1/2/3;
- большая таблица всех мест;
- таблица изменений рейтинга;
- кнопка “К шахматке”;
- кнопка “Поделиться”;
- кнопка “На главную”;
- стрелка назад возвращает к шахматке;
- системная кнопка назад возвращает к шахматке.

Текущий фикс: итоги снова landscape, чтобы не крашиться из-за смены ориентации.

## Готово / частично готово

- главный экран;
- база игроков;
- регистрация игрока;
- аватарка;
- медали;
- статистика соперников;
- создание турнира;
- выбор игроков из базы;
- круговой турнир до 10;
- шахматка;
- ввод результатов;
- W/L;
- RTTF-подобный рейтинг;
- бонусы за медали;
- история;
- экран итогов;
- фикс краша на итогах через landscape.

## Не готово

- полноценный режим 11+;
- группы;
- плей-офф;
- утешительная сетка;
- нормальная архитектура MVVM/Room;
- импорт/экспорт;
- облако.
