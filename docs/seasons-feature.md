# Сезоны клуба — техническая документация (реализовано)

> Версия: 2.0 · Дата: 2026-07-11  
> Оригинальное ТЗ сохранено в `docs/seasons-feature-v1-original.md`

---

## Текущее состояние

Система сезонов **полностью реализована** в `index.html` как часть одностраничного SPA на Firebase Realtime Database.

Реализовано:
- жизненный цикл сезона: `none → scheduled → active → awaitingCompletion → completed`;
- создание, запуск, мониторинг и завершение сезона через UI организатора;
- сезонный рейтинг, независимый от общего;
- блок сезона в разделе «Клуб» с динамическим порядком;
- экран сезона через `openSeasonScreen(seasonId)`;
- Игрок недели по последней завершённой календарной неделе (Europe/Moscow);
- idempotent-расчёт через Firebase transaction;
- Firebase Auth Email/Password + права через `admins/{uid}: true`.

---

## 1. Архитектура

Приложение — SPA в одном файле `index.html`.  
Хранилище — Firebase Realtime Database.  
Авторизация — Firebase Authentication (Email/Password).

### 1.1 Авторизация

Права организатора не определяются PIN-кодом или секретным словом.

```
adminPin     — удалён
secretWord   — удалён
```

Текущая схема:

```
Firebase Auth (Email/Password)
  ↓
signInWithEmailAndPassword()
  ↓
checkOrganizerAccess(uid)
  → db.ref(`admins/${uid}`).once('value') === true
  ↓
isAdmin = true
```

Запись `admins/{uid}: true` добавляется вручную в Firebase Console администратором проекта.  
Поддерживается несколько организаторов с разными UID.

### 1.2 Режимы доступа

| Режим | Условие | Доступ |
|---|---|---|
| Участник | без Auth | чтение данных клуба |
| Организатор | Auth + `admins/{uid}: true` | чтение + запись |

Административные кнопки скрыты от участников через `hidden` attribute + CSS `[hidden] { display: none !important }`.

## 2. Firebase Realtime Database — пути

### 2.1 Структура клуба

```
clubs/{clubId}/
  config           — метаданные клуба
  players          — JSON-строка массива игроков
  avatars/         — Base64-аватарки по playerId
  tournaments/     — завершённые турниры
  activeSeasonId   — ID текущего активного/запланированного сезона (или null)
  seasons/         — все сезоны клуба
  seasonStats/     — сезонная статистика по игрокам
  seasonRatingHistory/ — история сезонных изменений рейтинга
  seasonProcessedTournaments/ — маркеры обработанных турниров (idempotency)
  analytics/       — события аналитики (только участники)
```

### 2.2 Структура сезона

```
clubs/{clubId}/seasons/{seasonId}/
  id              — string (push-key)
  name            — string
  status          — 'scheduled' | 'active' | 'awaitingCompletion' | 'completed'
  startAt         — timestamp (ms)
  endAt           — timestamp (ms)
  completedAt     — timestamp (ms, только при completed)
  baseRating      — number (стартовый рейтинг, обычно 100)
  createdAt       — timestamp (ms)
  createdBy       — uid организатора
```

### 2.3 Сезонная статистика игрока

```
clubs/{clubId}/seasonStats/{seasonId}/{playerId}/
  rating          — текущий сезонный рейтинг
  matches         — матчи в сезоне
  wins            — победы в сезоне
  losses          — поражения в сезоне
  tournamentsPlayed — турниры в сезоне
```

### 2.4 История сезонного рейтинга

```
clubs/{clubId}/seasonRatingHistory/{seasonId}/{pushKey}/
  playerId        — string
  tournamentId    — string
  ratingBefore    — number
  ratingAfter     — number
  delta           — number
  timestamp       — timestamp (ms)
```

### 2.5 Idempotency-маркеры

```
clubs/{clubId}/seasonProcessedTournaments/{seasonId}/{tournamentId}
  — 'processing' | 'done' | 'error'
```

Записывается до начала расчёта через Firebase transaction.  
Повторный запуск расчёта для того же турнира в том же сезоне игнорируется.  
Источник повторного расчёта: `seasonCalculationSource`.

## 3. Жизненный цикл сезона

### 3.1 Состояния

```
none              — активного сезона нет (activeSeasonId отсутствует или null)
scheduled         — сезон создан, но startAt ещё не наступил
active            — startAt наступил, endAt не наступил
awaitingCompletion — endAt наступил, организатор ещё не завершил вручную
completed         — организатор нажал «Завершить сезон»
```

> Состояния `draft` не существует. Сезон либо запланирован (`scheduled`), либо активен.

Одновременно у клуба может быть только один сезон в состоянии `scheduled`, `active` или `awaitingCompletion`.

### 3.2 Переходы состояний

```
(нет сезона)
    ↓  организатор нажал «+ Начать сезон»
scheduled
    ↓  наступил startAt (автоопределение при загрузке)
active
    ↓  наступил endAt
awaitingCompletion
    ↓  организатор нажал «Завершить сезон»
completed
```

### 3.3 Создание сезона

Доступно только организатору.

Поля формы:

| Поле | Тип | Обязательное | Умолчание |
|---|---|---|---|
| Название | string | да | — |
| Дата начала | date | да | — |
| Дата окончания | date | да | — |
| Стартовый рейтинг | number | да | 100 |

При создании:
- всем текущим игрокам клуба присваивается `seasonStats[seasonId][playerId].rating = baseRating`;
- общий рейтинг игроков не изменяется;
- `activeSeasonId` устанавливается в ID нового сезона.

### 3.4 Новый игрок во время сезона

Если игрок добавлен после старта сезона:
- ему присваивается `rating = baseRating` в `seasonStats`;
- в расчёте участвует с первого же турнира сезона после добавления.

### 3.5 Завершение сезона

При нажатии «Завершить сезон»:
- `status` → `completed`;
- `completedAt` → текущий timestamp;
- `activeSeasonId` → `null`;
- сезонный рейтинг фиксируется и больше не изменяется;
- общий рейтинг не затрагивается.

Правило корректности: `startAt ≤ completedAt < endAt` (completedAt меньше planed endAt).

## 4. Расчёт сезонного рейтинга

### 4.1 Принцип

Расчёт запускается после каждого завершённого турнира, если:
- есть активный сезон;
- турнир попадает в даты сезона (`startAt ≤ tournament.completedAt < endAt`).

Используется тот же алгоритм, что и для общего рейтинга, но на входе — текущие **сезонные** рейтинги игроков, а не общие.

```
calculateGeneralRating(results, generalRatings)   ← прежний код
calculateSeasonRating(results, seasonRatings)     ← новый код (те же формулы)
```

### 4.2 Idempotency

Перед расчётом в Firebase transaction устанавливается маркер:

```
seasonProcessedTournaments/{seasonId}/{tournamentId} = 'processing'
```

После успешного расчёта: `'done'`.  
При ошибке: `'error'`.

Если маркер уже `'done'` — расчёт пропускается. Это исключает двойной пересчёт.  
Для принудительного повтора: изменить значение маркера через `seasonCalculationSource`.

### 4.3 Изоляция ошибок

Ошибка сезонного расчёта **не отменяет** сохранение турнира и пересчёт общего рейтинга.

```
saveTournament()          ← выполняется всегда
updateGeneralRating()     ← выполняется всегда

try {
  updateSeasonRating()    ← только при активном сезоне
} catch {
  logSeasonError()        ← фиксируется, не прерывает поток
}
```

## 5. Игрок недели

Игрок недели определяется по **последней полностью завершённой календарной неделе** (пн–вс) в часовом поясе `Europe/Moscow`.

Текущая (незавершённая) неделя не учитывается.

Локализация: `ru-RU`.  
Формат дат в UI: `'d MMMM'` (русский язык).

До первого турнира в клубе:
- блок «Игрок недели» **не показывает лидера**;
- медали мест (🥇🥈🥉) не отображаются.

## 6. Раздел «Клуб» — динамический порядок блоков

Вкладка «Клуб» в нижней навигации.

### 6.1 Состояние `none` (нет активного сезона)

```
РЕКОРДЫ КЛУБА
  [Самый титулованный]
  [Самый быстрый рост]
  [Лучшая серия]
  [Самый стабильный]

ИГРОК НЕДЕЛИ
  [синяя карточка]

ИГРОКИ С НАГРАДАМИ
  [список]
```

Организатор видит кнопку **«+ Начать сезон»** под заголовком.  
Участник видит текст _«Новый сезон скоро начнётся»_.

### 6.2 Состояние `scheduled`

```
РЕКОРДЫ КЛУБА
  [блоки]

ИГРОК НЕДЕЛИ
  [карточка]

ЗАПЛАНИРОВАННЫЙ СЕЗОН           ← после Игрока недели
  [Карточка: название, даты, обратный отсчёт до старта]

ИГРОКИ С НАГРАДАМИ
```

### 6.3 Состояние `active`

```
ТЕКУЩИЙ СЕЗОН                    ← перед Игроком недели
  [Карточка сезона]

РЕКОРДЫ КЛУБА
  [блоки]

ИГРОК НЕДЕЛИ
  [карточка]

ИГРОКИ С НАГРАДАМИ
```

### 6.4 Состояние `awaitingCompletion`

Аналогично `active`. Организатор видит кнопку **«Завершить сезон»**.

### 6.5 Состояние `completed`

```
РЕКОРДЫ КЛУБА
  [блоки]

ИГРОК НЕДЕЛИ
  [карточка]

ЗАВЕРШЁННЫЙ СЕЗОН                ← компактный итог после Игрока недели
  [Название, победитель, топ-3, кнопка «Открыть»]

ИГРОКИ С НАГРАДАМИ
```

## 8. Экран сезона

Открывается через:

```js
openSeasonScreen(seasonId)
```

Активный пункт нижней навигации при открытии: **«Клуб»** (не меняется).  
Возврат: кнопка «←» возвращает в раздел «Клуб».

Маршрут реализован как overlay поверх основного экрана (без изменения URL), а не как отдельная страница.

### 8.1 Шапка

```
← Сезон Осень 2026

1 сентября — 1 января · До конца: 47 дней
```

### 8.2 Лидер сезона

```
ЛИДЕР СЕЗОНА

[Аватар] Васильев Николай

126.7
1 место в сезоне

+4.2 за последний турнир
```

Показатели только по данным текущего сезона:

```
28 матчей  ·  15 побед  ·  3 турнира
```

> **Win streaks (серии побед)** перенесены на этап 2.

### 8.3 Топ сезона

```
[место] [аватар] Имя игрока    126.7   +4.2   >
```

- 🥇 золото — 1 место;
- 🥈 серебро — 2 место;
- 🥉 бронза — 3 место;
- остальные — серые цифры.

Общий рейтинг здесь не отображается.

### 8.4 Продвижение за неделю

Изменение сезонного рейтинга за последние 7 дней (из `seasonRatingHistory`).

```
Продвижение за неделю

Васильев Николай   +4.2   (зелёный)
Петров Петр        +3.1   (зелёный)
Иванов Иван        -1.4   (красный)
```

## 9. Профиль игрока

Существующий профиль не переработан.

Если есть активный сезон — добавляется компактный блок ниже карточек статистики:

```
Сезон Осень 2026                     >

126.7
1 место в сезоне

Старт: 100.0 · Матчей: 28
```

Блок кликабелен → открывает `openSeasonScreen(seasonId)`.

Если активного сезона нет — блок скрыт.

## 10. Работа без активного сезона

- турниры работают как раньше;
- общий рейтинг работает как раньше;
- история работает как раньше;
- профиль игрока без сезонного блока;
- раздел «Клуб» показывает рекорды и Игрока недели без карточки сезона.

## 12. Известные ограничения

1. **Win streaks в сезоне** не реализованы — перенесены на этап 2.
2. **Экран сезона** реализован как overlay, URL не меняется — глубокие ссылки на сезон невозможны.
3. **Шаринг сезона** (`Web Share API`) — базовый, без изображения-превью.
4. **Автоматический переход** `active → awaitingCompletion` происходит при следующей загрузке страницы или входе, а не в реальном времени.
5. **Firebase Security Rules** пока не обновлены под новую схему Auth — запись блокируется для участников на уровне Rules, но правила ещё не ограничивают запись по `admins/{uid}`.
6. **Аналитика** записывается только для участников; ошибки PERMISSION_DENIED подавлены.

## 13. Регрессионные проверки

Перед каждым релизом убедиться:

### Сезон отсутствует
- [ ] Открыть клуб как участник → раздел «Клуб» показывает рекорды без сезонного блока
- [ ] Создать и завершить турнир → общий рейтинг пересчитан, сезонный не трогался
- [ ] Профиль игрока → нет блока сезона

### Создание сезона
- [ ] Войти как организатор → кнопка «+ Начать сезон» видна
- [ ] Создать сезон с датой начала в будущем → статус `scheduled`
- [ ] Участник видит раздел «Клуб» с запланированным сезоном после Игрока недели
- [ ] Всем игрокам создана запись `seasonStats[seasonId][playerId].rating = baseRating`

### Активный сезон
- [ ] После наступления `startAt` → статус `active`, карточка сезона перемещается выше Игрока недели
- [ ] Завершить турнир → общий рейтинг обновлён, сезонный рейтинг обновлён
- [ ] Повторно завершить тот же турнир → идемпотентность: сезонный рейтинг не меняется
- [ ] `seasonProcessedTournaments[seasonId][tournamentId] = 'done'`

### Завершение сезона
- [ ] Нажать «Завершить сезон» → статус `completed`, `activeSeasonId = null`
- [ ] `completedAt` установлен, `completedAt < endAt`
- [ ] Сезонный рейтинг заморожен, общий рейтинг продолжает работать
- [ ] Завершённый сезон показывается как компактный итог в разделе «Клуб»

### Авторизация
- [ ] Войти как участник → кнопка «+» не видна, analytics-btn скрыт
- [ ] Войти как организатор → `isAdmin = true`, кнопки видны
- [ ] Logout → кнопки исчезают мгновенно, `auth.currentUser = null`
- [ ] `localStorage`: нет `tennis_admin_pin`, нет email/пароля

## 15. Принципиальные правила

> Общий рейтинг живёт в разделе **«Рейтинг»** и никогда не обнуляется.  
> Сезонный рейтинг живёт в разделе **«Клуб»** и стартует заново в каждом сезоне.  
> Сезоны — независимый слой поверх работающих турниров, рейтинга, профилей и наград.
