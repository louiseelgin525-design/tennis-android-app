# Известные проблемы и следующие задачи

## Проверить первым делом

1. Краш после просмотра результатов.
   Последний фикс: `FinalResults` снова `LANDSCAPE`.

2. Регистрация игрока.
   Проверить:
   - кнопка “Сохранить”;
   - поля не сжимаются;
   - камера;
   - галерея;
   - данные сохраняются.

3. Выбор игрока в турнир.
   Проверить:
   - только из базы;
   - однофамильцы;
   - аватарка маленькая;
   - ФИО и рейтинг не перекрываются.

4. Быстрый ввод счёта.
   Пользователь хочет максимально быстрый сценарий:
   - нажал матч;
   - выбрал нужный счёт;
   - окно закрылось.

5. Экран итогов.
   Нужно сохранить компактный дизайн:
   - маленькие плашки 1/2/3;
   - основное место под таблицу мест;
   - отдельно рейтинг;
   - возврат к шахматке.

## Архитектурная задача

Сейчас всё в одном `MainActivity.kt`.

После стабилизации желательно разделить:
- `models/`
- `screens/`
- `components/`
- `rating/`
- `storage/`
- `navigation/`
