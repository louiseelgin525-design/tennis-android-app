# Prompt для Jules

Ты продолжаешь Android-проект TennisTourney.

Используй актуальный файл:

```text
project_files/app/src/main/java/com/example/tennistourney/MainActivity.kt
```

Не используй старые версии из папки:

```text
previous_reference_versions_DO_NOT_USE_AS_CURRENT
```

## Первая задача

Сначала собрать проект. Если есть ошибки компиляции — исправить.

Потом проверить сценарий:

1. Регистрация 4 игроков.
2. Создание турнира.
3. Выбор игроков из базы.
4. Ввод результатов.
5. Завершение турнира.
6. Вопрос бонусов.
7. Экран итогов.
8. Возврат к шахматке.
9. На главную.
10. История.

## Главные правила

- Игрок не участвует, если его нет в базе.
- Рейтинг максимально близко к RTTF.
- W/L рейтинг не меняют.
- Медали сохраняются всегда.
- Бонусы за медали только по решению судьи.
- Интерфейс должен быть удобен на реальном Android-телефоне.

## На что обратить внимание

Пользователь особенно недоволен:
- сжатыми полями;
- огромными декоративными плашками;
- неудобным вводом счёта;
- крашами при переходе к итогам;
- тем, что однофамильцев сложно различать.

Нужно делать как Android-разработчик + UI/UX-дизайнер.
